Admin Login / Portal – Extremely Important First Step
What is the Login page?

This is the secure entry point to the entire Master Admin Panel.
Every time you want to manage content (collections, heritage, publications, exhibitions, enquiries, etc.), you must log in here first.
URL: https://tamilinformationcentere.org/admin/login
It protects your site — only people with correct email + password can access the admin area.

How to Log In – Step-by-Step (Very Important!)

Logging In Securely

  1. Open the login page
    Go to: https://tamilinformationcentere.org/admin/login
    (or click any admin link → it redirects here if not logged in)
  2. Enter Email Address
    In the "Email Address" field:
    Type your admin email exactly (e.g. admin@example.com)
    Case-sensitive — usually lowercase
    Double-check spelling — wrong email = cannot log in
  3. Enter Password
    In the "Password" field:
    Type your secret password
    • Password is hidden (dots or asterisks)
    • Case-sensitive
    • If forgotten → contact website owner/developer (no reset link shown)
  4. Click Sign In
    Click the big purple/blue Sign In button
    If correct → you enter the admin dashboard
    If wrong → shows red error like "Invalid credentials" — try again carefully

Important Security Rules – Read Carefully!

Never share your password — not with anyone, even friends/family/colleagues.
Use a strong password — at least 12 characters, mix letters/numbers/symbols.
Log out after use — especially on shared computers (click your name top-right → Logout).
Use computer/laptop — avoid phone/tablet (harder to type securely).
Check URL — always starts with https://tamilinformationcentere.org/admin — never click suspicious links.
If suspicious activity — change password immediately and inform owner.

What to do if login fails

  1. Double-check email and password spelling/case
  2. Make sure Caps Lock is off
  3. Try on different browser/device
  4. If still fails → contact website owner/developer (via WhatsApp/email) — they can reset or check account
  5. Never try random passwords many times — may lock account temporarily
Extra Tips for Safe Login:
  • Bookmark the login page — https://tamilinformationcentere.org/admin/login
  • Use password manager (e.g. browser built-in or LastPass) — never write it down
  • After login → check top-right: your name should appear (Admin)
  • Always log out when finished — click your name → Logout
  • If you see "Secured connection" lock icon — good (https)
Collection Section – Extremely Detailed Guide
What is the Collection section?

This is the heart of your admin panel. Here you create, view, edit, delete and organize all groups of content (collections) that appear on the public website.
Each collection is like a folder containing PDFs, images, descriptions — visitors see them under categories like "Human Rights & Justice", "Displacement & Recovery", etc.
Everything you do here directly affects what the public sees on tamilinformationcentere.org.

Collection has 4 main sub-sections.
Below are very long, step-by-step explanations for each one.

1. Add Collection – Create a Brand New Collection

What this section is for:

This is where you make a new "folder" or "album" of content.
One collection = one topic (example: all reports about refugees in one place).
After you create it and set it to Public, visitors on tamilinformationcentere.org can see and download the PDFs/images.

Very detailed step-by-step instructions:

  1. Find the menu
    Look at the left dark sidebar.
    Find the word Collection (it has a folder icon).
    Click the small arrow ▶ next to it to open sub-menu.
    Then click Add Collection (or sometimes it says "+ Add New Collection").
  2. Choose Main Category *
    This is a dropdown box (you click it and a list appears).
    Select one category that best matches your content.
    Examples: "Human Rights & Justice", "Governance & Institutions", "Displacement & Recovery".
    Why important? This decides under which section the collection will show on the public website.
    If no good category exists? First go to "Manage Category" (section 3 below) and create a new one.
  3. Collection Title *
    Type a clear name in this box.
    Good titles (examples):
    • Refugee Rights Reports 2024–2025
    • Conflict and Peace Building Documents
    • Media Coverage on Displacement
    Rules: Maximum 255 characters (about 40–50 words).
    Tip: Make it understandable even for someone who doesn't know Tamil.
  4. Title Image
    Click "Choose file" or "Browse".
    Select one beautiful cover photo from your computer.
    Recommended: size at least 800×500 pixels, JPG/PNG/WEBP format, max 5 MB.
    Why have a cover image? It makes the collection look attractive on the website homepage and category pages.
  5. Description *
    This is the biggest text box.
    Write everything visitors should know about this collection.
    Use the toolbar at the top to: • Make text bold or italic
    • Change colors
    • Add bullet points or numbered lists
    • Insert links (if needed)
    Good structure:
    1st paragraph: short summary (2–3 sentences)
    2nd–3rd paragraph: more details
    Last lines: what kind of documents are inside
  6. PDF Document
    Click "Choose file" → select your main PDF.
    Rules: max 50 MB, PDF format only.
    Important: If you set Access Type to Public, this PDF will be downloadable by everyone.
    Tip: If PDF is bigger than 50 MB, use free online compressor (smallpdf.com or ilovepdf.com) first.
  7. Access Type *
    Choose one radio button:
    • Public → anyone on the internet can see and download
    • Private → only logged-in admins or selected members can see
    Recommendation: Use Public for most collections unless the content is sensitive/confidential.
  8. Collection Images
    Click "Choose files" → select up to 10 photos from your computer.
    Rules: each image max 5 MB, formats JPG/PNG/GIF/WEBP.
    Best practice: Use photos that relate to the topic (maps, people, events, documents scans).
    These images will appear as a photo gallery on the collection page.
  9. Final step – Save
    Scroll all the way down.
    Check everything one last time.
    Click the large blue button Create Collection.
    What happens next?
    • Success message appears
    • You are usually taken to the new collection's detail page or back to list
    • If Public → it should now appear on the public website (refresh the site to check)
Extra Helpful Tips (please read):
  • Take 2 minutes to double-check everything before clicking Create
  • Save your work often (especially long descriptions) – copy to Notepad first if nervous
  • Use meaningful file names before upload (example: refugee-report-2025.pdf)
  • After creating → open the public website in new tab and search for your new collection to see how visitors will experience it
  • If upload fails → most common reasons: file too big / wrong format / internet slow
Things that can go wrong (and how to avoid):
  • Forgetting * fields → system shows red error and won't save
  • Uploading 20 MB image → fails – always check file size first
  • Setting to Private by mistake → nobody can see it on website
  • Very long title → looks bad on mobile phones
  • No images/PDF → collection looks empty and boring

What this section is for:

This is the place where you look at all the collections you (or others) have already created.
You can: - See a list of everything - Open any collection to see full details (what visitors see on the website) - Edit anything (change title, description, images, PDF, etc.) - Delete collections you no longer want

Most of your daily work will happen here — checking, updating, or removing old content.

    • Title → name of each collection

    • Created → date it was first made
    • Actions → 3 colored buttons/icons:
      • View full details
      • Edit / change anything
      • Delete forever
  4. How to use this list
    • Scroll up/down to see all collections
    • Type in search box → list gets smaller instantly
    • Click column headers (if clickable) to sort by title/date
    • Look at Access Type column — only Public ones are visible to website visitors

Part C – Edit Collection

Click yellow Edit button or pencil icon → opens almost same form as "Add Collection", but everything is already filled.

  1. Change basic info
    You can edit:
    • Main Category (change to another one)
    • Collection Title (make it better or fix spelling)
    • Description (add more text, change formatting)
  2. Update Title Image
    You see "Current Title Image" with preview
    • Click red trash icon to remove current image
    • Then click "Choose file" to upload new one
  3. Update PDF Document
    If PDF exists → shows current filename (e.g. report-2025.pdf)
    • Red trash → remove current PDF
    • "Choose file" → upload new PDF to replace it
  4. Manage Collection Images
    Shows all current images with small red trash icon under each one
    • Click trash on any image → removes it immediately
    • Scroll down to "Upload New Images" → choose multiple files (up to 10 total after delete/add)
    • You can keep adding until you reach max 10
  5. Change Access Type
    Switch between Public ↔ Private anytime
    Warning: Changing to Private hides it from public instantly
  6. Save your changes
    Scroll to bottom → click blue Update Collection
    After saving:
    • Success message appears
    • You return to detail page or list
    • Changes appear immediately on public website (if Public)

Part D – Delete Collection

Only delete when you are 100% sure — it is permanent.

  1. From list view → click trash icon () on any row
    OR from detail page → click red Delete Collection button
  2. Confirmation popup appears with title "Delete Collection"
  3. Message inside popup:
    "Are you sure you want to delete this collection?"
    Red warning text: "This action cannot be undone."
  4. Two buttons: • Gray Cancel → closes popup, nothing happens
    • Red Yes, Delete → permanently removes the collection
  5. What happens when you delete?
    • Collection disappears from admin list
    • All images, PDF, description are gone forever
    • If it was Public → immediately disappears from public website
    • No way to recover — even we (developers) cannot bring it back
Very serious warning:
Only click "Yes, Delete" if you really want to remove it completely.
Many admins regret deleting something useful — think twice!

What this section is for:

Every time you add or edit a collection, you must choose a "Main Category" from a dropdown.
This page is where you create, change or remove those category names.
Examples of categories you already have: "Conflict, Resistance & Peace", "Human Rights & Justice", "Media & Research", etc.

Step-by-step – Main Categories List Page

  1. How to open this page

  2. What you see at the top
    Page title: "Collection Categories"
    Small text: "Manage master categories for your collections"
    Two small cards: • Total Categories (e.g. 6) with icon

  3. Search box
    "Search categories by name..."
    Type any word (e.g. "rights") → list filters instantly
  4. The main table – All Categories
    Columns:
    • # – number 1, 2, 3...
    • Category Name – name with small icon (e.g. Conflict, Resistance & Peace)

Adding a New Category

  1. Click blue + Add Category button at top right
  2. Popup modal opens with title Add Category

    Type new name (must be unique – cannot repeat existing ones)
    Max 255 characters

  4. Click blue Add Category button
    OR Cancel if you change your mind
  5. After adding → popup closes, new category appears in the table instantly

Editing an Existing Category

  1. In the table → find the category → click pencil icon
  2. Popup opens: title Edit Category
  3. Field Category Name * shows current name
    Change it to something new (still must be unique)
  4. Click blue Save Changes
    OR Cancel to keep old name

Deleting a Category

  5. Buttons: Cancel or Yes, Delete
  6. If you click Yes → category disappears from table
  7. What really happens:
    • Category gone from dropdown forever
    • Any collections using this category may show blank category or error
    • Public website menus/dropdowns update instantly
Very important warning – do NOT delete categories unless necessary!
Deleting breaks organization. If you must remove one, first edit all collections using it to another category.

4. Collection Request – Manage User Access Requests for Private Collections

What this section is for:

When you create a collection and set it to Private, normal website visitors cannot see it.
Only logged-in authorized users (members/admins) can access it.

If a normal user wants to see a private collection, they can send an "access request" through the public website.
Those requests come here for you (Master Admin) to review, approve, or reject.

This section helps you control who gets special access to private/sensitive content.

How to open this page

  1. Look at the left dark sidebar menu.
    Find Collection (folder icon).
    Click the small arrow ▶ next to it to expand sub-menu.
  2. In the sub-menu, click Collection Request
    (sometimes labeled as "Collection Request" or "Access Requests")
  3. The page opens with title: Access Requests
    Subtitle: "Manage private collection access requests from users"

What you see on the main screen

  1. Top stats row (colored badges):
    • Total: total requests ever (e.g. 0)
    • Pending: requests waiting for your decision (orange)
    • Approved: you said yes (green)
    • Rejected: you said no (red)
  2. Filter buttons (below stats):
    • All (blue) – shows everything
    • Pending (orange icon) – only waiting ones
    • Approved (green check)
    • Rejected (red X)
  3. Search box:
    "Search name or email..."
    Type user name or email → list filters instantly
  4. Main content area:
    If no requests: big envelope icon + text "No requests found"
    "No access requests match the current filter."

What happens when there ARE requests

When users send requests from the public website, this page fills up with rows. Each row shows:

  • User's full name
  • Email address
  • Which private collection they want access to (title)
  • Date & time of request
  • Status: Pending / Approved / Rejected
  • Actions buttons (depending on status):
    • For Pending: Approve (green) / Reject (red)
    • For Approved/Rejected: maybe view details or undo (if system allows)

How to handle a Pending request – Step-by-step

  1. Find the request
    Use filter buttons → click Pending (orange)
    OR use search box if you know the user's name/email
  2. Read the request
    Look at: • User's name & email
    • Which collection they want
    • Date sent
    Decide: Should this person get access? (Is he a trusted member? Does he need this private content?)
  3. To Approve
    Click green Approve button on that row
    What happens:
    • Status changes to Approved (green)
    • User gets email notification (if system sends)
    • User can now log in and see the private collection
    • No more "Pending" for this request
  4. To Reject
    Click red Reject button
    Sometimes a popup asks for reason (e.g. "Not authorized" or "Invalid request")
    What happens:
    • Status changes to Rejected (red)
    • User may get email saying access denied
    • Request is closed
  5. After action
    Page refreshes → request moves from Pending to Approved/Rejected
    Stats badges at top update automatically
Helpful Tips for Managing Requests:
  • Check requests every few days — users are waiting
  • Only approve trusted people (members you know)
  • If unsure → ask the user via email before approving
  • Keep "Pending" count low — clear them regularly
  • If many requests for same collection → consider making it Public instead
Important Warnings:
  • Approving wrong person → private/sensitive content leaks to public
  • Rejecting everyone → users may get frustrated and stop visiting
  • No undo button for approve/reject (in most systems) — think carefully
  • If 0 requests always → maybe no private collections yet, or users don't know how to request

Heritage Collection – Extremely Detailed Guide
What is the Heritage Collection section?

This section is specially for cultural, historical, traditional, or heritage-related content.
It works almost exactly like the normal "Collection" section, but focused on old documents, photos, artifacts, traditions, audio-visual materials, etc.
Use this for things that are part of Tamil/Sri Lankan heritage, history, culture, or memory preservation.

Heritage Collection has 4 main sub-sections, just like normal Collection.

1. Add Heritage Collection – Create a New Heritage Item

What this does:

This creates a new heritage-focused collection.
It is very similar to "Add Collection", but meant for historical/cultural content.
Example: "Ancient Tamil Manuscripts", "Traditional Jewellery Photos", "War Memory Audio-Visuals".
Once Public, it appears in the heritage section of the website.

Very detailed step-by-step instructions:

  1. In left menu → click Heritage Collection (expand if needed)
    → click Add Heritage Collection (or "+ Add New Heritage Collection")
  2. Main Category *
    Choose from dropdown (heritage-specific categories).
    If none fit → go to "Manage Main Category" (section 3) and add one.
  3. Collection Title *
    Enter name (max 255 characters).
    Good examples: "Traditional Tamil Jewellery 1800–1950", "Memory of Conflict – Photos & Stories"
  4. Title Image
    Upload cover photo (JPG/PNG/WEBP – max 5MB).
    Use high-quality historical or cultural image.
  5. Description *
    Write full details using rich text editor (bold, lists, colors).
    Include historical context, importance, source if known.
  6. PDF Document
    Upload main PDF (max 50MB).
    Optional for private, recommended for public heritage items.
  7. Access Type *
    • Public → everyone can see
    • Private → only authorized users
  8. Collection Images
    Upload up to 10 extra photos (max 5MB each).
    Use for supporting visuals (scans, artifacts, etc.).
  9. Click blue Create Collection button at bottom.
    Success → new heritage collection created.
Tips for Heritage Collections:
  • Use very high-quality images — heritage content needs authenticity
  • Fill description with historical facts, dates, sources
  • PDF optional for private, but very useful for public
  • Fill all * fields — required

How to handle an enquiry – Step-by-step

  1. Find the enquiry
    Scroll table or use browser search (Ctrl+F) for name/email/subject
  2. View full details
    Click blue View button (or eye icon) on row
    Opens popup or new page showing: • Sender name & email • Subject • Full message text • Submission date/time • Possibly IP/location (if logged)
  3. Reply to the enquiry
    If system has reply button/form: • Click Reply (or similar) • Type response in editor • Send → status may auto-change to "Replied" If no built-in reply: • Copy email → open your email client • Reply manually → then update status here
  4. Update status
    Click dropdown in Status column (or in detail view):
    • Pending → Read / In Progress
    • In Progress → Replied / Resolved
    • Replied → Closed
    Save → table updates instantly
  5. Delete or archive (if needed)
    If option exists (trash icon in actions):
    Click → confirm "Are you sure?"
    Use sparingly — better to mark as Closed
Helpful Tips for Managing Enquiries:
  • Check daily — badge shows unread count (e.g. 12)
  • Reply within 24–48 hours — builds trust with visitors
  • For general enquiries → direct to relevant page/section
  • Spam or inappropriate → mark as Closed or delete
  • Keep records — note replies in status/comments if possible
  • If many enquiries → categorize (if system adds tags)
Important Warnings:
  • Never share sender email/phone publicly
  • Be professional & polite in replies — represents organization
  • If sensitive (complaints, personal info) → handle privately
  • Do NOT ignore Pending enquiries — users may follow up
  • Backup/export enquiries regularly (if export option exists)
